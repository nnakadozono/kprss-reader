# AGENT.md

## Project

`kprss-reader` is a static reader for article data generated by the sibling
`../kprss` project.

## Current approach

- Keep this project independent from `kprss`.
- Generate static JSON from the SQLite DB.
- Serve one static app from `public/`.
- Write generated output to `dist/`; do not commit `dist/`.
- Store reader state in browser `localStorage`.

## Important commands

```sh
cp .env.sample .env
python3 scripts/generate_site.py --out dist
python3 -m http.server 8000 --directory dist
```

## Implementation notes

- URL date routing uses `?date=YYYY-MM-DD`.
- The visible date input, URLs, and generated files use `YYYY-MM-DD`.
- If no date is provided, the app loads the latest date from
  `data/manifest.json`.
- The generator defaults to the latest 10 article dates. Use `--days` for a
  different window.
- Reading mode is selection-based: articles are checked by default, checked
  articles remain visible and expand;
  toggling reading mode off returns to all articles collapsed.
- Copy buttons copy Markdown for the current article or all currently visible
  articles. Keep the Clipboard API fallback for iPhone/Safari HTTP testing.
- Copied article URLs should be Markdown links whose label is the trailing URL
  ID, such as `[197930](https://...)`.
- Article footer actions are copy, collapse, and unselect. There is no
  read/unread or star feature in v1.
- Article images come from the configured or inferred asset table, usually as
  Dropbox URLs, and should be rendered lazily.
- Titles from the source DB may contain line breaks. Collapse them into ` | `
  during generation so desktop headline rows stay compact.
- Do not add a backend for reader state unless the product direction changes.
- Prefer small, dependency-free static code unless a real need appears.

## Data assumptions

The source DB path comes from `--db`, `.env`, or `KPDB`. The article table comes
from `--article-table`, `KPRSS_READER_ARTICLE_TABLE`, `KPSHORT`, or schema
inference. The asset table comes from `--asset-table`,
`KPRSS_READER_ASSET_TABLE`, or schema inference.

Daily JSON files are expected under:

```text
dist/data/YYYY-MM-DD.json
```
